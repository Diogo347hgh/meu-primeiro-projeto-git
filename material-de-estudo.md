# Material de Estudo - Versão Final

Conceito 1: Versionamento  
O versionamento é o processo de registrar e controlar as mudanças feitas em um projeto ao longo do tempo. Ele permite voltar a versões anteriores e entender o histórico de modificações.

Conceito 2: Repositório  
É o local onde os arquivos do projeto e todo o histórico de alterações ficam armazenados. Pode estar na sua máquina (local) ou em uma plataforma online como o GitHub (remoto).

Conceito 3: Commit
Representa uma versão salva do projeto. Cada commit inclui uma mensagem que descreve o que foi alterado e serve como um ponto no tempo que você pode revisitar.

Conceito 4: Branch (Ramificação)
É uma linha de desenvolvimento independente dentro do projeto. Permite criar novas funcionalidades ou testes sem afetar a versão principal.

Conceito 5: Merge (Mesclagem)  
É o processo de unir as mudanças de uma branch (ramo) de volta à branch principal, consolidando o trabalho feito de forma colaborativa.


# Discussão

Por que o commit é importante?
O commit é importante porque registra cada modificação feita no projeto, permitindo rastrear quem fez cada mudança, quando e por quê. Isso facilita a correção de erros, o aprendizado e a colaboração entre os membros da equipe.

Qual a vantagem do GitHub? 
O GitHub oferece um ambiente centralizado para armazenar, versionar e compartilhar códigos. Ele facilita o trabalho em equipe, a colaboração com pull requests, o controle de histórico e o backup seguro dos projetos, tornando o desenvolvimento mais organizado e eficiente.
